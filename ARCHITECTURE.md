# Архитектура приложения "Менеджер проектов"

## Компоненты приложения

### 1. Слой представления (Views)
- **LoginWindow** - Окно авторизации
- **MainWindow** - Главное окно с навигацией
- **ProjectsView** - Представление проектов
- **TasksView** - Представление задач
- **TeamView** - Представление команды
- **StatisticsView** - Представление статистики
- **ProjectDialog** - Диалог создания/редактирования проекта
- **TaskDialog** - Диалог создания/редактирования задачи
- **EmployeeDialog** - Диалог создания/редактирования сотрудника

### 2. Слой бизнес-логики (ViewModels)
- **MainViewModel** - Управление навигацией и главным окном
- **ProjectsViewModel** - Логика работы с проектами
- **TasksViewModel** - Логика работы с задачами
- **TeamViewModel** - Логика работы с сотрудниками
- **StatisticsViewModel** - Логика статистики
- **ProjectDialogViewModel** - Логика диалога проекта
- **TaskDialogViewModel** - Логика диалога задачи
- **RelayCommand** - Реализация команд для привязки

### 3. Слой данных (Models)
- **Project** - Модель проекта
- **Task** - Модель задачи
- **Employee** - Модель сотрудника

### 4. Слой доступа к данным (Data)
- **ProjectManagerDbContext** - Контекст Entity Framework для работы с БД

### 5. Слой сервисов (Services)
- **DatabaseService** - Сервис для транзакционной работы с БД

### 6. Конвертеры (Converters)
- **PriorityToColorConverter** - Конвертер приоритета в цвет
- **StatusToColorConverter** - Конвертер статуса в цвет

### 7. Ресурсы (Resources)
- **Styles.xaml** - Глобальные стили приложения
- **App.xaml** - Ресурсы приложения

## Основные алгоритмы

### Алгоритм авторизации
1. Запуск приложения → Открытие LoginWindow
2. Ввод логина и пароля
3. Валидация (проверка на пустоту)
4. Если валидация успешна → Открытие MainWindow
5. Если ошибка → Сообщение пользователю

### Алгоритм сохранения проекта
1. Открытие ProjectDialog
2. Ввод данных (название, описание, дедлайн)
3. Валидация (название обязательно)
4. Начало транзакции
5. Добавление/обновление в DbContext
6. SaveChanges()
7. Commit транзакции
8. Обновление списка проектов

### Алгоритм фильтрации задач
1. Пользователь выбирает критерии фильтрации
2. TasksViewModel получает событие изменения фильтра
3. Загрузка всех задач из БД (если еще не загружены)
4. Применение LINQ-фильтров (Where)
5. Обновление FilteredTasks
6. Автоматическое обновление View через привязку данных

### Алгоритм быстрой смены статуса
1. Правый клик на задачу → Контекстное меню
2. Выбор нового статуса
3. Начало транзакции
4. Обновление статуса задачи в БД
5. Commit транзакции
6. Обновление списка задач
